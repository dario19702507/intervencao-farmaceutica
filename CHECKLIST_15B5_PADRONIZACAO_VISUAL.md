# 15B.5 — Padronização visual para homologação

Este ajuste cria uma camada conservadora de coesão visual no frontend, sem alterar regras clínicas, chamadas de API ou fluxo de navegação.

## Arquivo alterado

- `frontend/src/style.css`

## Arquivo adicionado

- `CHECKLIST_15B5_PADRONIZACAO_VISUAL.md`

## O que foi padronizado

- Variáveis globais de cor, borda, raio, sombra e espaçamento.
- Tipografia geral, títulos, textos auxiliares e legibilidade.
- Cards, painéis, tabelas, formulários e campos.
- Botões primários, secundários, mini botões, botões de abas e botão de sair.
- Estados de foco, hover e disabled.
- Abas do prontuário, abas gerais e fluxo visual do consultório.
- Badges, chips, etiquetas de status e elementos de risco clínico.
- Responsividade básica para botões, cards e abas em telas menores.

## Validação manual recomendada

1. Abrir Dashboard.
2. Abrir Serviços Rápidos.
3. Abrir Consultório.
4. Abrir Pacientes.
5. Abrir Agenda.
6. Abrir Documentos.
7. Abrir Impressões.
8. Abrir Analytics.
9. Testar formulário com campos, botões, abas e tabela.
10. Verificar se os botões principais continuam funcionais.

## Observação

Este ajuste não corrige conteúdo textual, regras clínicas ou permissões. Ele apenas cria uma camada visual comum para reduzir diferenças entre telas antes do piloto multiusuário.
